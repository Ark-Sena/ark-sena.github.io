Inverse Rendering: A Case Study

This project documents the transformation of a physical object into a web-ready 3D asset, highlighting the inverse rendering pipeline.

1. The Subject: Cyberpunk 2077

I digitized a Johnny Silverhand figurine to push the pipeline to its limits. The challenge lay in the mix of materials: while matte clothing textures tracked perfectly, the reflective metallic arm and transparent glasses tested the robustness of the epipolar geometry and point matching.

2. Acquisition Pipeline

To ensure high-quality reconstruction from 322 images (Samsung S26 Ultra, 6120x8160 px), I maintained a strict workflow:

  • Controlled Capture: Constant focal length via turntable.
  • Data Density: >80% image overlap to guarantee triangulation stability.
  • Lighting: Diffuse interior setup to minimize parasitic reflections and harsh shadows.

3. Computational Reconstruction

The images were processed using RealityScan, following standard industry algorithms:

  • Structure from Motion (SfM): Estimating camera intrinsics/extrinsics and triangulating a sparse point cloud.
  • Multi-View Stereo (MVS): Pixel-depth inference for a dense point cloud.
  • Surface Reconstruction: Generating the mesh (Poisson algorithm) and re-projecting textures.

4. Optimization for Web

Raw scans are unusable for real-time applications without rigorous cleanup in Blender:

  • Cleanup: Removing non-target geometry (platform base).
  • Repair: Topological hole-filling where algorithms failed on reflective surfaces.
  • Decimation: Reducing polygon count for real-time web performance while preserving visual integrity.

5. WebGL Renders

The final optimized .glb asset is rendered in Three.js using a customized lighting environment, orbital controls, and material settings tailored for web display.
